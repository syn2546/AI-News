### DietrichGebert/ponytail 🔥 exploding
An AI agent plugin/skill (for Claude Code, Cursor, Copilot, Aider, etc.) that instructs models to write minimalist, platform-native code following the YAGNI principle—acting like the "laziest senior dev in the room" to reduce token usage and prevent over-engineering.
Stats: ⭐ ~67.0k · 🍴 ~3.4k · 📈 rapid · 🟢 active
Traction: Rapidly adopted by developers seeking to optimize token costs and code quality. Features sub-tools like `@ponytail-audit` and `ponytail-debt`. Author benchmarks claim up to a ~54% reduction in generated code lines, though Hacker News and Reddit communities are split between appreciating its efficacy and dismissing it as over-engineered prompting boilerplate.
References:

### baidu/Unlimited-OCR 🔥 exploding
An open-source, 3B-parameter Mixture-of-Experts (MoE) vision-language document-parsing framework that uses Reference Sliding Window Attention (R-SWA) to enable memory-efficient, single-pass OCR of long multi-page documents (40+ pages) without the KV cache bottleneck.
Stats: ⭐ ~12k · 🍴 ~941 · 📈 rapid · 🟢 active
Traction: Racked up 12k stars in its first week due to viral interest in solving the KV cache scaling bottleneck that plagues other end-to-end OCR models (like DeepSeek OCR). It achieves 93% accuracy on OmniDocBench v1.5. Key integrations are already under development, with vLLM adding support in late June 2026 and community efforts underway to connect it with Docling and llama.cpp.
References:
- https://github.com/baidu/Unlimited-OCR
- https://arxiv.org/abs/2606.23050
- https://huggingface.co/baidu/Unlimited-OCR

### HKUDS/CLI-Anything 🔥 exploding
An open-source framework and registry that transforms GUI-based, web, and API software into structured, agent-native CLI wrappers. This allows AI agents to control applications programmatically with deterministic JSON outputs rather than using fragile pixel-level GUI automation.
Stats: ⭐ ~44.3k · 🍴 ~4.1k · 📈 rapid · 🟢 active
Traction: High real-world interest driven by AI coding agents (such as Claude Code, Cursor, and Cline). Developers use these CLI wrappers to automate desktop apps without the token overhead or visual error rates of GUI agents. It acts as a lightweight alternative to MCP. The project also maintains its own package registry, CLI-Hub (clianything.cc), to list and distribute community wrappers.
References:
- https://github.com/HKUDS/CLI-Anything
- https://clianything.cc/
- https://arxiv.org/abs/2606.03854
